  <files>index.html</files>
  <action>
Add the following CSS rules near the top of the `<style>` block (after the CSS variables section, around line 162-170).

INSERT these rules immediately after the existing `*, *::before, *::after` rule:

```css
/* Mobile viewport fixes */
html {
    -webkit-text-size-adjust: 100%;
    text-size-adjust: 100%;
}

html, body {
    overflow-x: hidden;
    max-width: 100%;
}

img, video, iframe {
    max-width: 100%;
    height: auto;
}
```

<task type="auto">
  <name>Task 2: Add touch target minimum sizes</name>
  <files>index.html</files>
  <action>
Add the following CSS rules in the style block (can be added after the mobile viewport fixes from Task 1):

```css
/* Touch target accessibility - 44px minimum for tappable elements */
a, button, [role="button"],
input[type="submit"], input[type="button"], input[type="reset"],
.btn, .nav-link, .nav-cta, .mobile-toggle,
.year-tab, .package-card, .faq-question {
    min-height: 44px;
    min-width: 44px;
}

/* Ensure form inputs meet touch target and prevent iOS zoom */
input, textarea, select, button {
    font-size: 16px; /* Explicit 16px prevents iOS auto-zoom on focus */
    min-height: 44px;
    touch-action: manipulation; /* Removes 300ms tap delay */
}

/* Improve tap feedback on mobile */
@media (hover: none) and (pointer: coarse) {
    a, button, .btn {
        -webkit-tap-highlight-color: rgba(124, 58, 237, 0.2);
    }
}
```

IMPORTANT: Do NOT modify the existing `.form-input, .form-textarea, .form-select` rules. The new rules will work alongside them.
  </action>

Grep for "min-height: 44px" - should appear in touch target rules.
Grep for "touch-action: manipulation" - should appear for inputs.
Grep for "font-size: 16px" with explicit px value for form elements.
  </verify>
  <done>
All interactive elements have 44px minimum touch targets.
Form inputs have explicit 16px font-size (prevents iOS auto-zoom).
Touch action manipulation removes tap delay.
  </done>
</task>
